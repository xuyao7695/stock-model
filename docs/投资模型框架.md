# 股票投资模型框架 v0.1

> **目标**：把你的临场交易决策，固化为可记录、可复盘、可约束的规则系统。

---

## 一、模型定位

| 维度 | 立场 |
|---|---|
| 选股信号 | **外部**（大V/社群推荐） + **规则筛选**（风控 + 自身胜率过滤） |
| 决策方式 | **纪律优先**（硬规则 > 临场判断） |
| 收益目标 | 持续稳定（年化 10-20%） > 短期高弹 |
| 风险偏好 | 严控单笔回撤、严控总回撤 |

**核心信念**：模型的首要价值不是"选到更多牛股"，而是**把你不擅长的"自我控制"外包给代码**。

## 三、数据规范

### 3.1 交易日志字段（**向前记录 + OCR 回填统一**）

| 字段 | 类型 | 说明 | 来源 |
|---|---|---|---|
| `date` | YYYY-MM-DD | 交易日期 | 券商对账单 |
| `time` | HH:MM | 交易时间 | 券商对账单 |
| `action` | enum | 证券买入 / 证券卖出 / 卖出 / 申购配号 / 银行转取 | OCR |
| `name` | string | 标的名称 | OCR |
| `code` | string | 标的代码（待补） | 人工补全 |
| `price` | float | 成交价 | OCR |
| `qty` | int | 数量 | OCR |
| `amount` | float | 卖出=成交额；买入=成本 | OCR |
| `signal_source` | enum | 大V / 社群 / 自选 / 指标 | **人工补** |
| `buy_reason` | text | 买入理由（标的/价位/逻辑） | **人工补** |
| `target_price` | float | 目标卖出价 | **人工补** |
| `stop_loss` | float | 止损位 | **人工补** |
| `max_hold_days` | int | 最大持仓天数 | **人工补** |
| `emotion` | enum | FOMO/恐慌/贪婪/平静 | **人工补** |
| `followed_plan` | bool | 是否按计划执行 | **人工补** |
| `deviation_note` | text | 偏差原因 | **人工补** |

> **关键**：只有"信号来源、买入理由、目标/止损/最大持仓、情绪、是否按计划"5 个字段是**必须人工记录**的，模型离开了它们就只是一堆买卖流水。

### 3.2 持仓/汇总数据

- 月度 P&L 来自对账单"区间盈亏"行
- 累计 P&L = sum(各月 P&L)
- 同期上证 = 同期上证指数涨跌幅

## 四、规则引擎

### 4.1 默认风控参数（可在 `screener/rules.json` 修改）

| 规则 | 默认值 | 含义 |
|---|---|---|
| `max_single_position` | 30% | 单只票最大仓位 |
| `max_total_position` | 80% | 总仓位上限（保留 20% 现金） |
| `daily_loss_circuit_breaker` | -5% | 单日累计亏损达 -5% 即停手 |
| `max_single_loss` | -8% | 单笔硬止损 |
| `min_risk_reward` | 2.0 | 期望收益 / 最大亏损 ≥ 2 |
| `max_daily_trades` | 3 笔 | 日内超过 3 笔即不再交易 |
| `max_holding_days` | 10 天 | 持仓超过 10 天强制复盘 |

### 4.2 每日 Checklist（盘前/盘中）

- [ ] 总仓位 ≤ 80%？
- [ ] 单一标的 ≤ 30%？
- [ ] 今日已亏损 ≥ 5%？（是 → 停手）
- [ ] 拟买入的盈亏比 ≥ 2:1？
- [ ] 今日已交易笔数 < 3？
- [ ] 拟买入有明确止损位？
- [ ] 信号来源过去 30 日胜率 ≥ 40%？
- [ ] 非追涨（已涨 > 5% 慎入）？
- [ ] 非情绪化（写一句买入理由）？

### 4.3 买入决策流程

```
信号 (大V/社群)
    ↓
信号胜率过滤（来源历史胜率 < 40% → 拒绝）
    ↓
候选名单 (含代码/价位/逻辑)
    ↓
风控闸门（仓位/熔断/盈亏比/笔数）
    ↓
    ├─ 通过 → 写入"待买入"清单（带止损位）
    └─ 拒绝 → 记录拒绝原因
```

### 4.4 卖出决策流程

```
持仓票
    ↓
触发检查：到目标价? 到止损? 持仓天数? 大V反转信号?
    ↓
    ├─ 任一触发 → 立即执行
    └─ 未触发 → 每日复盘清单
```

## 五、行为复盘指标

### 5.1 核心指标

| 指标 | 公式 | 健康线 |
|---|---|---|
| 胜率 | 盈利单数 / 总卖出单数 | ≥ 50% |
| 盈亏比 | avg(盈利) / abs(avg(亏损)) | ≥ 2.0 |
| 盈利因子 | sum(盈利) / abs(sum(亏损)) | ≥ 1.0 |
| 月度胜率 | 盈利月数 / 总月数 | ≥ 60% |
| 跑赢上证 | 月 P&L 是否 > 月指数收益 | ≥ 60% 月份 |
| 持仓周期 | 卖出日 - 买入日 | 中位 ≥ 3 天 |
| 换手强度 | 总交易笔数 / 标的数 | ≤ 3 笔/股 |

### 5.2 行为偏差检测

- **砍盈扛亏（处置效应）**: `avg(盈利单持仓天数) - avg(亏损单持仓天数) > 1` → 警告
- **过度交易**: 日均 > 3 笔 → 警告
- **追涨杀跌**: 买入时股价已涨 > 5% → 标记
- **FOMO 集中**: 单日买入 ≥ 3 只且无理由 → 警告

### 5.3 月度复盘清单

- [ ] 月 P&L vs 上证
- [ ] 5 笔最大亏损的共性
- [ ] 5 笔最大盈利的共性
- [ ] 信号来源胜率排名（淘汰 < 40% 的）
- [ ] 持仓超 10 天的票
- [ ] 情绪化交易次数
- [ ] 风控规则违反次数
- [ ] 下月规则调整（如有）

## 七·补、实盘选股与风控规则（v0.2 新增）

### 7.1 选股扫描逻辑（`screener/scan.py` + `scan_config.json`）

**两条路径并行，结果合并按分排序：**

| 路径 | 数据源 | 核心条件 | 适用场景 |
|---|---|---|---|
| **A-连板** | 涨停池 `stock_zt_pool_em` | 连板数 1-5、封单 ≥ 3000万、炸板 0 次、流通市值 30亿-1000亿、行业热度 | 龙头/连板战法（主） |
| **B-强势** | 全市场 `stock_zh_a_spot_em` | 涨幅 3-9.4%、换手 ≥ 3%、量比 ≥ 1.5 | 技术突破+量能（限流时跳过） |

**题材热度算法**：涨停池按行业聚合 → 涨停数最多的行业热度=1.0，其余按比例归一化。

**评分公式**：
```
score = 0.30 × min(连板数/5, 1)
      + 0.25 × min(封板资金/5亿, 1)
      + 0.25 × 行业热度
      + 0.20 × 技术强势分（路径 B 才有）
```

### 7.2 风控规则（`screener/rules.json`，可调）

| 规则 | 默认值 | 触发动作 |
|---|---|---|
| `max_single_position` | 30% | 单只仓位超限 → 拦截 |
| `max_total_position` | 80% | 总仓位超限 → 拦截 |
| `daily_loss_circuit_breaker` | -5% | 当日累计亏损达 -5% → 全天停手 |
| `max_single_loss` | -8% | 单笔硬止损（到价无条件砍） |
| `min_risk_reward` | 2.0 | 盈亏比 < 2 → 不出手 |
| `max_daily_trades` | 3 | 当日交易笔数达 3 → 停手 |
| `max_holding_days` | 10 | 持仓超 10 天 → 强制复盘 |

### 7.3 交易纪律（写入每只候选的建议中）

每只候选的操作建议都附带以下 6 条纪律（`advise.py` 生成）：

1. ⛔ 单只 ≤ 建议仓位，总仓 ≤ 80%
2. 🛑 硬止损 -8%（到价无条件砍，**不补仓**）
3. 🎯 目标 +16% 或 题材退潮即走
4. ⏱ 最大持仓 10 天，到期强制复盘
5. 📝 买入前写一句理由 + 信号来源（**不写不买**）
6. 🔥 今日已亏 -5% 熔断 → 关 APP

### 7.4 仓位分级（按连板高度）

| 连板数 | 建议仓位 | 逻辑 |
|---|---|---|
| 首板（1板） | 18% | 轻仓埋伏，试错 |
| 二板 | 25% | 主仓位，题材确认 |
| 三板 | 20% | 半仓以内，防高位 |
| 四板及以上 | 15% | 高位轻仓试错，止损更紧（-6%） |

### 7.5 风控闸门流程（`advise.py` 的 `risk_gate`）

```
候选 → ① 日亏熔断检查 → ② 总仓位检查 → ③ 单只仓位检查 → ④ 日笔数检查
                                                                   ↓
                                                          ├─ 全通过 → ✅ 关注买入
                                                          └─ 任一失败 → ⛔ 风控拦截（附原因）
```

### 7.6 三端呈现

| 端 | 文件 | 特点 | 启动 |
|---|---|---|---|
| 企业微信机器人 | `delivery/wecom_bot.py` | 每日推送精简版建议到群，手机实时看 | `python run_daily.py`（自动推送） |
| Streamlit 看板 | `dashboard/app.py` | 浏览器交互，可筛选/排序/展开详情 | `streamlit run dashboard/app.py --server.port 8501` |

> **推荐组合**：日常用企业微信机器人推提醒（被动接收），需要细看时开 Streamlit 看板（主动分析），手机临时查看用 Flask App。
